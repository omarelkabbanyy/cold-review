# Project Input Template

Copy this underneath the Cold Review prompt. Short, concrete answers are better
than polished marketing language.

```text
Project:

Project type:
[Business / App / Product / Service / Content / Other]

Target user:

Problem:

How the project works:

Why someone would switch from their current solution:

Current alternatives or competitors I know:

Evidence:
[Users, payments, repeated use, interviews, waitlist, analytics, research, or “none”]

Feedback:
[Paste positive, negative, or contradictory feedback]

Constraints:
[Time, money, skills, tools, geography, regulations, deadline]

Decision I need to make:
[Kill / Pivot / Test / Build, or a specific decision]
```
